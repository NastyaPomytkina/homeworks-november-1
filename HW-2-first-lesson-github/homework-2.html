git config --global user.name blabla - ввод имени пользователя git config --global user.email blabal@bla - ввод эмайла пользователя git init - создает пустой репозиторий git status - проверяет новые или измененные файлы git add - добавляет файлы в коммит. Используем с "." или "*" чтоб добавить все файлы, чтоб добавить один файл пишем его имя и расширение git commit - выполняем коммит изменения(текущий слепок проэкта нужно сохранить) git log перечисляет коммиты, сделанные в репозитории в обратном к хронологическому порядке — последние коммиты находятся вверху. git checkout - позволяет перемещаться между ветками, созданными командой git branch git branch - перечисляет ветки, создает новые, удаляет и переименовывает их (менеджер веток). git push (origin master) - пушит или отправляет в репозиторий слепки проэкта git pull - извлекает и загружает содержимое из удаленного репозитория и немедленно обновляет локальный репозиторий этим содержимым. git merge - объединяет несколько последовательностей коммитов в общую историю. git clone - Клонирование локального или удаленного репозитория. Клонирование чистого репозитория. .gitignore определяет и защищает от случайного добавления в репозиторий файлов, которых там видеть не хотим Advanced git revert - выясняет какие изменения нужно отменить, отменяет их и создаёт новый коммит, в котором эти изменения проведены. git reset - команда, используемая для отмены локальных изменений в репозитории Git. git rebase - переносит изменения из одной ветки в другую, сжимает все изменения в один «патч» затем интегрирует патч в целевую ветку. git cherry-pick - берёт изменения, вносимые одним коммитом, и пытается повторно применить их в виде нового коммита в текущей ветке. git stash - помещает текущие изменения, которые выполнили в файлах, в отдельное хранилище, и возвращает файлы к исходному состоянию.
